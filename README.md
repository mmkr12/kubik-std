# KUBIK.std — Производство онлайн

Простая веб-платформа учёта заказов для рекламно-производственной компании KUBIK.
Next.js 14 (App Router) + TypeScript + TailwindCSS + shadcn/ui-style компоненты + Framer Motion + Supabase.

## Структура проекта

```
app/
  page.tsx                 — лендинг (hero, о компании, работы, калькулятор, контакты)
  production/page.tsx      — публичная страница "Производство онлайн" (все активные заказы)
  production/[id]/page.tsx — карточка заказа для клиента (эскиз, дата, статус)
  login/page.tsx           — вход для менеджеров
  admin/zamery/            — раздел "Замеры"
  admin/production/        — раздел "Производство" (кнопка ГОТОВО)
  admin/history/           — раздел "История" (поиск)
  admin/settings/          — раздел "Настройки" (цены, коэффициенты, КП)
  api/calculate/           — расчёт стоимости
  api/kp/                  — генерация коммерческого предложения (печатается в PDF)
components/                — UI-компоненты и админ-виджеты
lib/                       — supabase-клиенты, типы, расчёт цены, утилиты
supabase/schema.sql        — SQL-схема, RLS-политики, storage bucket'ы
supabase/seed.sql          — тестовые данные (необязательно)
```

## 1. Настройка Supabase

1. Создайте проект на [supabase.com](https://supabase.com).
2. В **SQL Editor** выполните файл `supabase/schema.sql` — он создаст:
   - таблицу `orders` (заказы на всех стадиях: замер → производство → история);
   - таблицу `calculator_settings` (цены, коэффициенты, подарки, текст КП);
   - политики RLS (клиенты видят только активные/готовые заказы, менеджеры — всё);
   - storage bucket'ы `sketches` и `finished-photos`.
3. (Необязательно) выполните `supabase/seed.sql`, чтобы добавить тестовые заказы.
4. Создайте пользователя-менеджера: **Authentication → Users → Add user** (email + пароль).
   Именно этими данными менеджер будет входить на `/login`.
5. Скопируйте `Project URL` и `anon public key` из **Project Settings → API**.

## 2. Настройка проекта

```bash
cp .env.example .env.local
```

Заполните `.env.local`:

```
NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key
```

Установите зависимости и запустите локально:

```bash
npm install
npm run dev
```

Откройте [http://localhost:3000](http://localhost:3000).

## 3. Деплой на хостинг (рекомендуется Vercel)

1. Загрузите этот проект в GitHub-репозиторий.
2. На [vercel.com](https://vercel.com) → **Add New Project** → выберите репозиторий.
3. В **Environment Variables** добавьте те же `NEXT_PUBLIC_SUPABASE_URL` и `NEXT_PUBLIC_SUPABASE_ANON_KEY`.
4. Нажмите **Deploy**.

Проект также работает на любом хостинге с поддержкой Node.js (Next.js standalone):
`npm run build && npm run start`.

## 4. Как пользоваться

- **Лендинг (`/`)** — клиенты рассчитывают стоимость и переходят в "Производство онлайн".
- **`/production`** — публичная страница, видно всем: карточки активных заказов, отсортированные по ближайшей дате монтажа.
- **`/production/[id]`** — открывается по клику на карточку, показывает клиенту только эскиз, дату и статус "В работе".
- **`/login`** — вход для менеджера/монтажника (учётная запись из Supabase Auth).
- **`/admin/zamery`** — менеджер добавляет заявку; кнопки "Согласовали" (→ в производство) / "Потеряли" (→ архив).
- **`/admin/production`** — та же очередь, что видят клиенты, но с полными данными; кнопка "ГОТОВО" открывает окно загрузки фото готовой работы и переносит заказ в историю.
- **`/admin/history`** — поиск по названию компании / телефону, фото готовых работ.
- **`/admin/settings`** — цены калькулятора, коэффициенты, подарки, текст и срок действия КП.

## 5. Коммерческое предложение (КП)

`/api/kp` генерирует красиво оформленную HTML-страницу с расчётом, сроками,
гарантией, QR-кодом и предложением, действующим 12 часов — она автоматически
открывает системное окно печати, так что клиент (или менеджер) может сохранить
её как PDF через "Сохранить как PDF" в диалоге печати браузера.

Если нужен PDF-файл, генерируемый на сервере без участия пользователя, эту
функцию можно расширить с помощью библиотеки `@react-pdf/renderer` или
headless-браузера (Puppeteer) — текущая реализация нарочно оставлена лёгкой,
без зависимости от headless-браузера на хостинге.

## 6. Дизайн

Палитра: белый, светло-серый градиент (`bg-mist-gradient`), тёмно-синий
(`navy-900`), синий градиент (`bg-blue-gradient`). Везде — карточки, без таблиц,
вдохновение — Apple / Stripe / Linear.
